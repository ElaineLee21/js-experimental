UI, API

UI(User Interface)
: 사용자가 시스템을 제어하기 위해서 사용하는 조작장치 ex)button

API(Application Programming Interface)
: 애플리케이션을 만들기 위해 프로그래밍할 때 사용하는 조작장치 ex)alert



-Final advice

-프로젝트를 할 때, 필수불가결한 최소한의 도구만으로 문제를 해결해보라. 모든 개념을 총동원하려고 하지 말 것



-지금까지 배운 것을 적용하다가 한계에 봉착했을 때 쓸만한 검색어

document객체

태그 삭제하고 싶거나 자식태그 추가하고 싶을 때 document 객체를 살펴보자.
필요한 method가 있을 것이다.

DOM(Document Object Model)

그래도 안되면 DOM(Document Object Model)을 찾아보자. 좀더 넓은 개념이다.

window객체

웹페이지가 아니라 웹브라우저 자체를 제어해야 한다면 window객체를 조사해보자.
ex)현재 열려있는 웹페이지 주소 알아내기, 새 창 열기, 웹브라우저 화면크기 JS통해 알아내기
- window객체에 써있는 property나 method가 나를 도울 것이다.

ajax

웹페이지를 리로드하지 않고 정보를 변경하고 싶다면 ajax.
현대적인 웹앱을 만들기 위한 필수적 테크닉이다.

cookie

웹페이지가 리로드 되어도 현재상태를 유지하고 싶다면 cookie를 배우자.
사용자를 위한 개인화된 서비스를 제공할 수 있다.

offline web application

인터넷이 끊겨도 동작하는 웹페이지를 만들고 싶다면 offline web application을 찾아보자

webRTC

화상통신 웹앱을 만들고 싶다면 webRTC를 찾아보자

speech

사용자의 음성을 인식하고, 음성으로 정보를 전달하고 싶다면 speech로 시작하는 API를 살펴보자

webGL

3차원 그래픽으로 게임같은 것을 만들고 싶다면 webGL

webVR

가상현실에 관심이 많다면 webVR
